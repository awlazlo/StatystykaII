Wstęp

Chociaż przyczyny rozwoju raka piersi nie są dokładnie znane, istnieje kilka czynników, które mogą zwiększać ryzyko jego wystąpienia. Do najbardziej istotnych zalicza się czynniki dziedziczne, które znacznie zwiększają prawdopodobieństwo choroby. Oprócz czynników genetycznych znaczacą rolę odgrywa płeć (u mężczyzn zdarza się 100 razy rzadziej), wiek, liczba potomstwa, alkohol oraz otyłość. Badania dowodzą, że na zwiększenie ryzyka wystąpienia raka piersi mają wpływ również czynniki środowiskowe. Najczęściej rak piersi występuje w Ameryce Północnej i krajach Europy Zachodniej, czyli w krajach wysoko uprzemysłowionych. Najrzadziej natomiast w Japonii i Chinach.

W naszych zbiorze danych, dla każdego przypadku zachorowania dysponujemy takimi informacjami jak: płeć, przedział wiekowy oraz powiat. Wykorzystując ostatnią informację postanowiłyśmy dodać do modelu zmienną ciągłą stopień urbanizacji, uzyskaną ze strony GUS. Ponieważ dla każdego przypadku choroby znamy powiat występowania, mogłyśmy także dodać wyżej wymieniane czynniki, takie jak np. liczba potomstwa czy stopień otyłości. Wiązałoby się to jednak z koniecznością przypisania do każdego przypadku wielkości uśrednionej dla całego powiatu, co mogłoby być niemiarodajne oraz zaburzać wyniki, dlatego ostatecznie pozostawiłyśmy trzy zmienne: płeć, przedział wiekowy oraz stopień urbanizacji.

Aby otrzymać miarodajne dane dla każdego obszaru, znormalizujemy je za pomocą danych dotyczących ludności w poszczególnych powiatach, pochodzące ze strony GUS.

Analiza różnych modeli

Do przeprowadzenia analizy postanowiłyśmy wykorzystać metodę walidacji krzyżowej. Pozwala ona lepiej ocenić trafność prognostyczną modelu, niż zwykły podział na próbę testową i uczącą.

Regresja liniowa

mse_lm <- function(){
    smpl <- createDataPartition(dane_pred$nowy, p=0.8)[[1]]
    train <- dane_pred[smpl,]
    test <- dane_pred[-smpl,]
    model_lm <- lm(nowy~.,data=train)
    return(mse(test$nowy,predict(model_lm, newdata=test)))
}

#mse_lm()

Model ridge

mse_ridge <- function(){
    smpl <- createDataPartition(dane_pred$nowy, p=0.8)[[1]]
    train <- dane_pred[smpl,]
    test <- dane_pred[-smpl,]
    model_ridge <- cv.glmnet(as.matrix(train[,-3]),as.matrix(train$nowy), alpha=0)
    p <-predict(model_ridge,newx=as.matrix(test[,-3]))
    return(mse(p,as.matrix(test$nowy)))
}

#mse_ridge()

#mse_lasso()

Klasyfikacja lda

mse_lda<-function() {
  smpl <- createDataPartition(dane_pred$nowy, p=0.8)[[1]]
    train <- dane_pred[smpl,]
    test <- dane_pred[-smpl,]
    
    classification_lda<-lda(nowy~.,data=train)
    p<-predict(classification_lda,newdata=test[,-3])$class
    predi<-as.numeric(as.vector(p))
    
    return(mse(as.matrix(predi),as.matrix(test$nowy)))
    
}

#mse_lda()

Naive Bayes

mse_Bayes<-function() {
  smpl <- createDataPartition(dane_pred$nowy, p=0.8)[[1]]
    train <- dane_pred[smpl,]
    test <- dane_pred[-smpl,]
    
    classification_Bayes<-naiveBayes(as.factor(nowy)~.,data=train)
    p<-predict(classification_Bayes,test[,-3])
    predi<-as.numeric(as.vector(p))
    
   return(mse(predi,as.matrix(test$nowy)))
}

Random forest

mse_rf<-function() {
  smpl <- createDataPartition(dane_pred_temp$V3, p=0.8)[[1]]
    train <- dane_pred_temp[smpl,]
    test <- dane_pred_temp[-smpl,]
    
    classification_rf<-randomForest(V3~.,data=train,ntree=20)
    prediction<-predict(classification_rf,newdata=test[,-3])
    
    return(mse(prediction,test$V3))
}

#mse_rf()

Porównanie MSE wszystkich metod

Funkcję skonstruowaną powyżej wywołałyśmy 100 razy, w celu obliczenia błędu MSE dla każdej metody. Wyniki powyższej operacji przedstawione są na poniższym wykresie:

wynik_lm<-data.frame(replicate(100, mse_lm()))

wynik_ridge<-data.frame(replicate(100, mse_ridge()))

wynik_lasso<-data.frame(replicate(100, mse_lasso()))

wynik_lda<-data.frame(replicate(100, mse_lda()))

wynik_Bayes<-data.frame(replicate(100, mse_Bayes()))

wynik_rf<-data.frame(replicate(100, mse_rf()))

wynik_total<-bind_rows(wynik_lm,wynik_ridge,wynik_lasso,wynik_lda,wynik_Bayes,wynik_rf)
ggplot(wynik_total,aes(x=model,MSE))+geom_violin(aes(fill=model))+ stat_summary(fun.data = "mean_cl_boot", colour = "black", size = 0.7)+theme_bw()

Na wykresie widać, że metoda Random Forest osiąga niższe błędy niż pozostałe metody, dlatego tą metodę wybrałyśmy do predykcji.
